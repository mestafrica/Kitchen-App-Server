KitchenApp

General notes

API for the MEST Kitchen Online Ordering System (MKOO)

API detail

BreakfastOptions

BreakfastOptions, LunchOptions and SupperOptions are used to create the menu for each day.

A BreakfastOption object specifies a particular meal and a date.

A query can be made for the BreakfastOptions for a particular date. The returned list can be used to create a menu of choices, from which a user can select one as their breakfast order for that day.

Create BreakfastOption

Create a breakfast option

  • Curl
  • HTTP
curl -X POST -H "Content-Type: application/json" -d '{
	"breakfast_option" : {
		"serving_date": "Wed Apr 19 2017",
		"meal_id": 1
	}
}' "https://mest-kitchen-app.herokuapp.com/v1/breakfast_options"
POST /v1/breakfast_options HTTP/1.1
Host: mest-kitchen-app.herokuapp.com
Content-Type: application/json

{
	"breakfast_option" : {
		"serving_date": "Wed Apr 19 2017",
		"meal_id": 1
	}
}

Read BreakfastOption

Fetch a BreakfastOption by it’s id

  • HTTP

GET /v1/breakfast_options/1 HTTP/1.1
Host: mest-kitchen-app.herokuapp.com

Menu

The menu endpoint does not reference an actual menu object in the database. Instead, it is a sort of shorthand for performing operations on a list of breakfast_options, lunch_options and supper_options for a particular day at a go.

NB: There is no endpoint for updating a menu. Instead you should delete the menu, and create it again

Create menu

Create breakfast_options, lunch_options and supper_options for a particular date at a go.

The date should be in the format yyy-mm-dd, e.g. “2017-05-01”

The breakfast_options, lunch_options, and supper_options are specified as an array of meal ids as shown below

{
	"menu": {
	  "serving_date": "2017-05-01",
	  "breakfast_choices": [3,4],
	  "lunch_choices": [1,2],
	  "supper_choices": [5,6]
	}
}

Note that they are called breakfast_choices, lunch_choices and supper_choices not breakfast_options, lunch_options and supper_options as you would normally expect.

  • Curl
  • HTTP
curl -X POST -H "Content-Type: application/json" -d '{
	"menu": {
	  "serving_date": "2017-05-01",
	  "breakfast_choices": [3,4],
	  "lunch_choices": [1,2],
	  "supper_choices": [5,6]
	}
}' "https://mest-kitchen-app.herokuapp.com/v1/menus"
POST /v1/menus HTTP/1.1
Host: mest-kitchen-app.herokuapp.com
Content-Type: application/json

{
	"menu": {
	  "serving_date": "2017-05-01",
	  "breakfast_choices": [3,4],
	  "lunch_choices": [1,2],
	  "supper_choices": [5,6]
	}
}

Read menu

Get the menu for a particular date.

The date is specified in the url. It should be in this format yyy-mm-dd, e.g “2017-05-01”.

The returned object is a list of breafast_options, a list of lunch_options and a list of supper_options for the specified date.

  • Curl
  • HTTP
curl -X GET "https://mest-kitchen-app.herokuapp.com/v1/menus/2017-05-01"
GET /v1/menus/2017-05-01 HTTP/1.1
Host: mest-kitchen-app.herokuapp.com

Read menus

Get the menu for a range of dates

The start_dates and end_dates are specified in the url. They should be in this format yyy-mm-dd, e.g “2017-05-01”.

The returned object is a list of breafast_options, a list of lunch_options and a list of supper_options for each day between the start_date and end_date specified (inclusive).

This endpoint is mostly used for getting the menu for a particular week

  • Curl
  • HTTP
curl -X GET "https://mest-kitchen-app.herokuapp.com/v1/menus/2017-05-01/2017-05-07"
GET /v1/menus/2017-05-01/2017-05-07 HTTP/1.1
Host: mest-kitchen-app.herokuapp.com

Delete Menu

Delete the menu for a particular date.

The date is specified in the url. It should be in this format yyy-mm-dd, e.g “2017-05-01”.

This will delete all the underlying breakfast_options, lunch_options and supper_options for that date.

  • Curl
  • HTTP
curl -X DELETE "https://mest-kitchen-app.herokuapp.com/v1/menus/2017-05-01"
DELETE /v1/menus/2017-05-01 HTTP/1.1
Host: mest-kitchen-app.herokuapp.com
